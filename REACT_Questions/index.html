What's Virtual Dom in React & What are the advantages of it? (2 Marks)

Answer :

In React, the Virtual DOM (Document Object Model) is a lightweight representation of the actual DOM. It is a programming concept that React uses to efficiently update and render components. When you create a React application, you define the structure and behavior of the user interface using React components. Each component has a render method that describes what the component should render based on its current state and props.When the state or props of a component change, React updates the Virtual DOM instead of directly manipulating the actual DOM. React then performs a diffing algorithm to determine the minimal number of changes needed to update the Virtual DOM to match the new component state. After the diffing process, React generates a list of changes, or "diffs," and applies these changes to the actual DOM, resulting in an efficient update to the user interface. This approach helps avoid unnecessary and costly manipulations of the DOM.

Advantages of the Virtual DOM in React:

  • Performance
  • Efficiency
  • Cross-platform compatibility
  • Simplified programming model

Explain LifeCycle of React Components? (1 Marks)

Answer :

In React, components have a lifecycle that consists of different phases or stages. These lifecycle phases allow developers to execute code at specific points during a component's existence. React provides a set of methods, known as lifecycle methods, which can be overridden to define custom behavior at each phase.The lifecycle of a React component can be divided into three main phases: mounting, updating, and unmounting.In addition to that lifecycle methods, React also provides some other lifecycle methods that are less commonly used or have specific use cases, such as error handling (componentDidCatch()) and controlling behavior when props change (componentDidUpdate() and getSnapshotBeforeUpdate()).

Question :

Whats the difference between between Functional Components and Class Components? (1 Marks)

Answer :

The Main Difference:

  • Syntax:
    • Functional components are defined as functions, while class components are defined as ES6 classes.

  • State:
    • Functional components do not have their own state. They rely on props passed from parent components. Class components have their own state, which can be managed using the setState() method.

  • Lifecycle methods:
    • Class components have lifecycle methods like componentDidMount(), componentDidUpdate(), etc., which allow developers to perform actions at specific stages of the component's lifecycle. Functional components can use hooks like useEffect() to achieve similar functionality.

  • Readability and simplicity:
    • Functional components are generally considered more readable and simpler to write and understand. They have a more straightforward structure and do not require managing complex class-based syntax.

What are the hooks in React & Can we use Hooks in Class Components? (2 Marks)

Answer :

In React, hooks are functions that allow you to use state and other React features in functional components. They were introduced in React 16.8 as a way to write stateful logic without the need for class components.

Regarding the use of hooks in class components, hooks are specifically designed for functional components and are not intended to be used in class components directly. Hooks rely on the functional component's ability to manage state and lifecycle with greater simplicity.

In either case, it's important to note that class components and functional components have different lifecycles and syntax, so the migration or integration of hooks may require careful consideration and adjustment of the code.

Question :

Explain useEffect & Advantages of it (2 Marks)

Answer :

The useEffect hook is a built-in hook in React that allows functional components to perform side effects and manage lifecycle behavior. It replaces the functionality of lifecycle methods like componentDidMount, componentDidUpdate, and componentWillUnmount in class components.The useEffect hook takes two arguments: a function (effect) and an optional array of dependencies. The effect function is executed after every render, and it can perform side effects such as data fetching, subscriptions, or interacting with the DOM. The dependencies array is used to specify values that the effect depends on. If any of the dependencies change, the effect is re-run.

Advantages of the useEffect hook:
  • Side effects in functional components
  • Lifecycle management
  • Cleanup and resource management
  • Integration with other hooks
